奇怪的python整数

发表于 2017-08-30 | 分类于 Algorithm and Computer Science

现象1

可以看到，所有值相同的‘小整数’在同一块内存区域。使用id深入研究：

看起来对于小整数有专门的表／数组存放，每个整数占据32个字节。

原因：
出于性能考虑，python将小整数对象缓存，保存到名为small_ints的链表中。在python整个生命周期内，对需要引用这些整数对象的地方，不再重新创建对象，直接引用缓存中的对象。凡是[-5, 256]区间的小对象就从small_ints中取。

现象2

is和==区别：

  • is比较两对象的id值是否相等，即比较两对象是否为同一实例对象，是否指向同一个内存地址
  • ==比较两对象的内容是否相等，默认调用__eq__()方法
