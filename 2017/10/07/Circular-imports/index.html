Circular imports

发表于 2017-10-07 | 分类于 Algorithm and Computer Science

执行first.py, 因为循环导入出错：

Analysis

记住以下几点：

  • ‘import’和’from xxx import yyy’都是可执行语句。它们在读到那行时会被执行；
  • 如果一个模块不在sys.modules中，import语句会在sys.modules中创建新的模块入口，然后执行那个模块。直到执行完才会把’control’还给’calling module’；
  • 如果模块已经在sys.modules中了，import语句只会简单的返回那个模块（无论是否已经执行完）。这就是为什么循环导入可能返回部分空的模块；
  • 最后一点，python some_file.py 的时候，some_file.py 自己也成为 module，并且这个 module 的名字叫 __main__，如果 some_file.py 直接或者间接地导入 some_file.py 自己，则被导入的 some_file.py 会成为一个新的 module，且这个新的 module 不是原来的 __main__。

Solutions

1、把`from x import y’移到后面

2、改成import模块本身
